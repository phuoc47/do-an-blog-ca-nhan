Xử lý ngoại lệ trong Java

  • bởi admin
  • 18/10/2025
1. Ngoại lệ là gì?

Trong lập trình, ngoại lệ (Exception) là những lỗi xảy ra trong khi chương trình đang chạy, khiến luồng thực thi bị gián đoạn. Ví dụ: chia cho 0, đọc file không tồn tại, hoặc truy cập phần tử mảng ngoài phạm vi.

int ketQua = 10 / 0; // Lỗi ArithmeticException
2. Cấu trúc try - catch cơ bản

Để tránh chương trình bị dừng đột ngột, ta sử dụng khối try-catch để “bắt” ngoại lệ và xử lý phù hợp.

try {
    int ketQua = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Không thể chia cho 0!");
}

Trong ví dụ trên, chương trình không bị lỗi dừng, mà thay vào đó in ra thông báo thân thiện cho người dùng.

3. Sử dụng nhiều khối catch

Bạn có thể bắt nhiều loại ngoại lệ khác nhau bằng nhiều khối catch.

try {
    int[] arr = new int[2];
    arr[5] = 10;
} catch (ArithmeticException e) {
    System.out.println("Lỗi số học!");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("Truy cập vượt quá kích thước mảng!");
} catch (Exception e) {
    System.out.println("Lỗi không xác định: " + e.getMessage());
}
4. Khối finally

Khối finally luôn được thực thi dù có xảy ra ngoại lệ hay không – thường dùng để đóng tài nguyên như file hoặc kết nối cơ sở dữ liệu.

try {
    FileReader fr = new FileReader("data.txt");
} catch (FileNotFoundException e) {
    System.out.println("Không tìm thấy file!");
} finally {
    System.out.println("Đóng tài nguyên.");
}
5. Tự tạo ngoại lệ bằng throw

Java cho phép bạn chủ động ném (throw) ngoại lệ khi gặp tình huống không hợp lệ.

public static void kiemTraTuoi(int tuoi) {
    if (tuoi < 18) {
        throw new IllegalArgumentException("Chưa đủ 18 tuổi!");
    }
    System.out.println("Đủ tuổi tham gia!");
}

Đoạn code trên sẽ chủ động thông báo lỗi nếu người dùng nhập tuổi không hợp lệ.

6. Khai báo ngoại lệ với throws

Nếu một phương thức có thể phát sinh ngoại lệ, ta cần khai báo bằng từ khóa throws để các lớp khác biết và xử lý.

public void docFile() throws IOException {
    FileReader file = new FileReader("data.txt");
}
7. Tạo ngoại lệ tùy chỉnh (Custom Exception)

Để rõ ràng và có tính mô-đun cao, bạn có thể tự định nghĩa ngoại lệ riêng cho ứng dụng của mình.

class TuoiKhongHopLeException extends Exception {
    public TuoiKhongHopLeException(String message) {
        super(message);
    }
}

public class Demo {
    static void kiemTra(int tuoi) throws TuoiKhongHopLeException {
        if (tuoi < 18)
            throw new TuoiKhongHopLeException("Tuổi không hợp lệ!");
        else
            System.out.println("Được phép truy cập!");
    }

    public static void main(String[] args) {
        try {
            kiemTra(16);
        } catch (TuoiKhongHopLeException e) {
            System.out.println(e.getMessage());
        }
    }
}
8. Lời khuyên thực tế
  • Không “nuốt” ngoại lệ bằng catch trống – luôn ghi log hoặc thông báo rõ ràng.
  • Chỉ bắt những ngoại lệ bạn có thể xử lý.
  • Dùng finally hoặc try-with-resources để đảm bảo tài nguyên được giải phóng.
  • Tạo custom exception cho các module lớn để dễ bảo trì.

Tóm lại: Xử lý ngoại lệ không chỉ giúp chương trình an toàn hơn mà còn thể hiện tư duy chuyên nghiệp của lập trình viên Java. Hãy luôn chuẩn bị cho “tình huống xấu nhất” và viết code dự phòng khéo léo!
