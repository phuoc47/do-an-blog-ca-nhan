Thành thạo hàm trong JavaScript

  • bởi admin
  • 12/10/2025
1. Giới thiệu về hàm

Hàm (Function) trong JavaScript là **trái tim của ngôn ngữ**. Chúng cho phép bạn nhóm các đoạn mã có liên quan lại với nhau để tái sử dụng, giúp mã ngắn gọn và dễ bảo trì hơn. JavaScript cung cấp nhiều cách khai báo hàm – mỗi loại có ưu và nhược điểm riêng.

2. Hàm khai báo (Function Declaration)

Đây là cách khai báo truyền thống và phổ biến nhất.

function chao(ten) {
    console.log("Xin chào, " + ten + "!");
}
chao("Phước"); // Kết quả: Xin chào, Phước!

Ưu điểm của loại hàm này là **hoisted** – tức bạn có thể gọi nó trước khi định nghĩa.

3. Hàm biểu thức (Function Expression)

Được gán cho một biến, và không được hoisted.

const tong = function(a, b) {
    return a + b;
};
console.log(tong(3, 5)); // Kết quả: 8

Hàm biểu thức hữu ích khi bạn muốn truyền hàm như một giá trị (ví dụ trong callback).

4. Hàm mũi tên (Arrow Function)

Giới thiệu trong ES6, cú pháp ngắn gọn và trực quan hơn.

const nhanDoi = x => x * 2;
console.log(nhanDoi(6)); // 12

Hàm mũi tên không có this riêng, rất tiện khi dùng trong các callback của sự kiện hoặc Promise.

5. Phạm vi (Scope) và Closure

Mỗi hàm trong JavaScript tạo ra một phạm vi riêng, giúp biến bên trong không ảnh hưởng đến bên ngoài. Khi một hàm con truy cập biến từ hàm cha, nó tạo thành **closure** – một tính năng mạnh mẽ giúp lưu trữ trạng thái.

function taoBoDem() {
    let dem = 0;
    return function() {
        dem++;
        console.log("Giá trị hiện tại: " + dem);
    };
}
const boDem = taoBoDem();
boDem(); // 1
boDem(); // 2
6. Callback và hàm bậc cao

JavaScript cho phép truyền hàm như một đối số – điều này tạo nên **lập trình bất đồng bộ** và **xử lý sự kiện linh hoạt**.

function xuLy(ham) {
    console.log("Bắt đầu xử lý...");
    ham();
}
xuLy(() => console.log("Đã hoàn thành!"));
7. Tổng kết

Hiểu và sử dụng thành thạo hàm trong JavaScript giúp bạn:

  • Tối ưu cấu trúc mã.
  • Giảm lặp lại logic.
  • Làm chủ lập trình bất đồng bộ (async/await, promise).
Hãy luyện tập viết nhiều loại hàm khác nhau và thử kết hợp chúng trong các bài toán thực tế – đó là con đường ngắn nhất để trở thành **JavaScript Developer chuyên nghiệp**.
