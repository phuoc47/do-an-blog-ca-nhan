Giới thiệu về Collections Framework

Java Collections Framework (JCF) là một tập hợp các lớp và giao diện giúp bạn lưu trữ, truy cập, và thao tác dữ liệu một cách linh hoạt. Thay vì phải tự cài đặt cấu trúc dữ liệu như mảng, danh sách, cây hoặc bảng băm, Java đã cung cấp sẵn cho lập trình viên một hệ thống chuẩn, mạnh mẽ và dễ mở rộng.

1. Cấu trúc tổng thể

Hệ thống Collections trong Java gồm 3 nhóm chính:

  • List – Lưu trữ các phần tử có thứ tự, cho phép trùng lặp.
  • Set – Lưu trữ các phần tử duy nhất, không trùng lặp.
  • Map – Lưu trữ dữ liệu theo cặp key-value.
2. List – Danh sách có thứ tự

List là lựa chọn lý tưởng khi bạn muốn lưu dữ liệu có thể trùng lặp và duy trì thứ tự thêm phần tử. Hai lớp phổ biến nhất là ArrayList và LinkedList. 

import java.util.*;

public class ViDuList {
    public static void main(String[] args) {
        List<String> danhSach = new ArrayList<>();
        danhSach.add("Java");
        danhSach.add("Python");
        danhSach.add("C#");
        danhSach.add("Java");

        for (String item : danhSach) {
            System.out.println(item);
        }
    }
}

Kết quả: 

Java
Python
C#
Java

ArrayList nhanh khi truy cập ngẫu nhiên (theo chỉ số), trong khi LinkedList hiệu quả hơn khi thêm/xóa phần tử ở giữa danh sách.

3. Set – Tập hợp không trùng lặp

Set được dùng khi bạn muốn đảm bảo mỗi phần tử chỉ xuất hiện một lần. Phổ biến nhất là HashSet và TreeSet. 

import java.util.*;

public class ViDuSet {
    public static void main(String[] args) {
        Set<String> tapHop = new HashSet<>();
        tapHop.add("Java");
        tapHop.add("Python");
        tapHop.add("Java"); // bị bỏ qua

        for (String item : tapHop) {
            System.out.println(item);
        }
    }
}

TreeSet sắp xếp phần tử theo thứ tự tự nhiên (hoặc comparator tùy chỉnh), trong khi HashSet tập trung vào tốc độ truy xuất.

4. Map – Bản đồ khóa/giá trị

Map khác biệt vì nó lưu dữ liệu theo cặp key → value. Mỗi khóa là duy nhất, và bạn có thể truy xuất giá trị bằng khóa. 

import java.util.*;

public class ViDuMap {
    public static void main(String[] args) {
        Map<Integer, String> hocVien = new HashMap<>();
        hocVien.put(1, "Phước");
        hocVien.put(2, "Nam");
        hocVien.put(3, "Huy");

        for (Map.Entry<Integer, String> entry : hocVien.entrySet()) {
            System.out.println(entry.getKey() + " - " + entry.getValue());
        }
    }
}

Ngoài HashMap, ta có thể dùng TreeMap (sắp xếp theo khóa) hoặc LinkedHashMap (duy trì thứ tự thêm phần tử).

5. Collections Utility Class

Lớp Collections cung cấp nhiều hàm tĩnh hữu ích như: 

Collections.sort(list);
Collections.reverse(list);
Collections.shuffle(list);
Collections.max(list);
Collections.min(list);

Những phương thức này giúp thao tác danh sách nhanh hơn và nhất quán, thay vì tự viết thuật toán.

6. Khi nào nên dùng loại nào?
  • ArrayList: Khi cần truy cập nhanh theo chỉ số.
  • LinkedList: Khi cần thêm/xóa thường xuyên.
  • HashSet: Khi cần loại bỏ phần tử trùng lặp nhanh chóng.
  • TreeSet: Khi cần tập hợp có sắp xếp.
  • HashMap: Khi cần tra cứu giá trị theo khóa với hiệu suất cao.
7. Tổng kết

Java Collections Framework là công cụ cốt lõi giúp quản lý dữ liệu hiệu quả, linh hoạt và dễ bảo trì. Hiểu đúng và chọn đúng cấu trúc sẽ giúp mã nguồn của bạn gọn gàng, dễ đọc và tối ưu hơn trong mọi dự án.
